# VK-бот для сообщества «АвтоКлуб НСК»

Простой и расширяемый бот на **Python 3.11+** для сообщества VK.
Использует **vk_api**, **Bots Long Poll API**, **SQLite** и переменные окружения через **.env**.

## Что умеет текущая версия

- Слушает входящие сообщения сообщества VK через Long Poll.
- Реагирует на триггеры: `старт`, `start`, `начать`, `привет`.
- Отправляет сообщения по кнопкам:
  - «Что входит»
  - «Как оплатить»
  - «Я оплатил»
  - «Партнёры»
  - «Задать вопрос»
- Сохраняет пользователей в локальную БД `database.db` при первом сообщении.
- Создаёт заявки на кнопку «Как оплатить», переводит в `paid` на кнопку «Я оплатил».
- Принимает скрин оплаты (вложения `photo`/`doc`) и отмечает факт получения в заявке.
- Отправляет пользователю ссылку в закрытый клуб после подтверждения админом `/approve {vk_id}`.
- Выдаёт доступ на 30 дней и поддерживает продления доступа ещё на 30 дней.
- Поддерживает админ-команды:
  - `/admin`
  - `/addpartner | Название | Категория | Скидка | Адрес | Телефон | Описание`
  - `/partners`
  - `/partneroff {id}`
  - `/partneron {id}`
  - `/users`
  - `/requests`
  - `/pending`
  - `/approve {vk_id}`
  - `/referrals`
  - `/refuser {vk_id}`
  - `/active`
  - `/expired`
  - `/user {vk_id}`
  - `/stats`

## Структура проекта

- `main.py` — запуск бота, обработка событий Long Poll и админ-команд.
- `db.py` — работа с SQLite (`users`, `requests`).
- `texts.py` — все тексты бота.
- `keyboards.py` — VK-клавиатура.
- `.env.example` — пример переменных окружения.
- `requirements.txt` — зависимости.

## 1) Как создать токен сообщества VK

1. Откройте сообщество VK, где будет работать бот.
2. Перейдите: **Управление → Работа с API**.
3. Во вкладке **Ключи доступа** создайте ключ типа сообщества.
4. Выдайте права минимум на `messages`.
5. Скопируйте токен и сохраните его в `.env` как `VK_GROUP_TOKEN`.

## 2) Как включить Bots Long Poll

1. В сообществе перейдите: **Управление → Работа с API → Long Poll API**.
2. Включите Long Poll API.
3. Во вкладке **Типы событий** включите минимум событие **Входящее сообщение** (`message_new`).
4. Узнайте ID сообщества (число) и добавьте его в `.env` как `VK_GROUP_ID`.

## 3) Администратор

Добавьте в `.env` ID администратора сообщества и ссылку на закрытый клуб:

```env
ADMIN_ID=123456789
CLUB_INVITE_LINK=https://vk.me/join/...
```

Этот пользователь сможет выполнять админ-команды внутри диалога бота:

- `/admin` — открыть кнопочную админ-панель «Админ-панель АвтоКлуб НСК».
  - Кнопка **«📊 Статистика»** выполняет `/stats`.
  - Кнопка **«🕓 Ожидают проверки»** выполняет `/pending`.
  - Кнопка **«✅ Активные»** выполняет `/active`.
  - Кнопка **«❌ Истёкшие»** выполняет `/expired`.
  - Кнопка **«📋 Все заявки»** выполняет `/requests`.
  - Кнопка **«🤝 Партнёры»** выполняет `/partners`.
- `/addpartner | Название | Категория | Скидка | Адрес | Телефон | Описание` — добавить партнёра.
- `/partners` — список партнёров в формате `id | name | category | discount | is_active`.
- `/partneroff {id}` — отключить партнёра (`is_active = 0`).
- `/partneron {id}` — включить партнёра (`is_active = 1`).
- `/users` — количество пользователей.
- `/requests` — список заявок в формате `vk_id | status | receipt_received | created_at | approved_at | access_until`.
- `/pending` — заявки в `paid`, которые ещё не подтверждены, плюс подсказка `Подтвердить: /approve {vk_id}` для каждой заявки.
- `/approve {vk_id}` — отметить последнюю заявку пользователя как `approved`, записать время подтверждения и отправить ссылку.
  - Если активный доступ уже есть (`access_until` в будущем), срок продлевается ещё на 30 дней (`is_renewal = 1`).
  - Если активного доступа нет, выставляется новый срок на 30 дней от текущего момента (`is_renewal = 0`).
- `/referrals` — топ рефереров в формате `referrer_vk_id | approved_count`.
- `/refuser {vk_id}` — карточка реферальных данных пользователя:
  - `referral_code`
  - `referrer_vk_id`
  - число подтверждённых приглашённых (`approved_referrals`)
- `/active` — пользователи с активным доступом в формате `vk_id | access_until`.
- `/expired` — пользователи с истёкшим доступом в формате `vk_id | access_until`.
- `/user {vk_id}` — подробная карточка пользователя и последней заявки.
- `/stats` — общая статистика по пользователям и заявкам.

## 4) Процесс оплаты

1. Пользователь нажимает кнопку **«Как оплатить»** → бот создаёт заявку `new`.
2. Пользователь нажимает **«Я оплатил»** → заявка переводится в `paid`.
3. Пользователь отправляет скрин оплаты (`photo`/`doc`) → бот ставит `receipt_received = 1`.
4. Админ подтверждает оплату командой `/approve vk_id`.
5. Бот отправляет пользователю приветствие, `CLUB_INVITE_LINK` и дату окончания доступа (`access_until`).
6. При повторном подтверждении активного участника доступ продлевается на 30 дней.
7. Если пользователь пришёл по реферальному коду, при подтверждении оплаты его рефереру засчитывается подтверждённое приглашение.

## Реферальная система

- У каждого пользователя есть персональный код `AC{vk_id}` (например `AC123456`).
- Пользователь может нажать кнопку **«Моя ссылка»** и получить:
  - ссылку вида `https://vk.com/im?sel=-{VK_GROUP_ID}&ref=AC{vk_id}`
  - резервный вариант: текстовый код и инструкцию для друга: `СТАРТ AC{vk_id}`.
- Бот распознаёт сообщения в формате `СТАРТ AC123456` и `start AC123456`.
- Если код валиден и не принадлежит самому пользователю, у приглашённого фиксируется `referrer_vk_id` (только один раз).
- При установке реферера создаётся запись в таблице `referrals`.
- Когда админ подтверждает приглашённого через `/approve {vk_id}`, в `referrals.approved_at` проставляется время подтверждения.

## 5) База данных

Бот использует SQLite-файл `database.db` (создаётся автоматически).

### Таблица `users`

- `id` — `INTEGER PRIMARY KEY AUTOINCREMENT`
- `vk_id` — `INTEGER UNIQUE`
- `first_name` — `TEXT`
- `last_name` — `TEXT`
- `created_at` — `TEXT`
- `last_activity_at` — `TEXT` (время последнего сообщения пользователя)
- `last_message_sent_at` — `TEXT` (время последнего автоматического напоминания)
- `referrer_vk_id` — `INTEGER` (vk_id пригласившего пользователя)
- `referral_code` — `TEXT UNIQUE` (персональный реферальный код `AC{vk_id}`)

### Таблица `requests`

- `id` — `INTEGER PRIMARY KEY AUTOINCREMENT`
- `vk_id` — `INTEGER`
- `status` — `TEXT` (`new` / `paid` / `approved`)
- `created_at` — `TEXT`
- `receipt_received` — `INTEGER` (`0` / `1`)
- `approved_at` — `TEXT`
- `access_until` — `TEXT` (дата окончания доступа)
- `is_renewal` — `INTEGER` (`0` / `1`, признак продления)

При старте бота выполняется безопасная миграция: если новых колонок ещё нет, они добавляются через `ALTER TABLE`.

### Таблица `partners`

- `id` — `INTEGER PRIMARY KEY AUTOINCREMENT`
- `name` — `TEXT NOT NULL`
- `category` — `TEXT`
- `discount` — `TEXT`
- `address` — `TEXT`
- `phone` — `TEXT`
- `description` — `TEXT`
- `is_active` — `INTEGER DEFAULT 1`
- `created_at` — `TEXT`

Если в БД есть активные партнёры, кнопка **«Партнёры»** показывает их список с категориями, скидками, адресами и телефонами. Если активных партнёров нет — бот отправляет текущий fallback-текст.

### Таблица `referrals`

- `id` — `INTEGER PRIMARY KEY AUTOINCREMENT`
- `referrer_vk_id` — `INTEGER`
- `referred_vk_id` — `INTEGER UNIQUE`
- `created_at` — `TEXT`
- `approved_at` — `TEXT`

## 8) Партнёры и скидки

Для управления каталогом партнёров доступны админ-команды:

- `/addpartner | Название | Категория | Скидка | Адрес | Телефон | Описание`
  - Пример: `/addpartner | Автомойка Drive | Автомойка | -20% | ул. Ленина, 10 | +7... | Скидка на комплексную мойку`
- `/partners`
- `/partneroff {id}`
- `/partneron {id}`

## 7) Автоматические напоминания

В боте запущен фоновый планировщик (`scheduler.py`) на `threading + time.sleep`, который раз в 10 минут проверяет условия автоворонки:

- **Не оплатил**: последняя заявка в `new` старше 2 часов.
- **Отправил скрин, но не подтверждён**: `paid + receipt_received = 1`, без `approved_at`, старше 1 часа.
- **Доступ скоро закончится**: `access_until` наступит в ближайшие 2 дня.
- **Доступ истёк**: прошло не менее 1 дня после `access_until`.

Защита от спама:

- напоминания не отправляются чаще 1 раза в 24 часа;
- используется поле `users.last_message_sent_at`.

При любом сообщении пользователя обновляется `users.last_activity_at`.
Планировщик работает в отдельном потоке и не останавливает Long Poll цикл бота даже при ошибках в фоновой задаче.

## Примечания

- Бот работает через Long Poll (без webhook).
- Платежи не автоматизированы: подтверждение выполняет администратор.
- Для отправки сообщений используется `random_id`, как требует `messages.send`.
- Не храните реальные токены в репозитории.
