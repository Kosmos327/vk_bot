# VK-бот для сообщества «АвтоКлуб НСК»

Простой и расширяемый бот на **Python 3.11+** для сообщества VK.
Использует **vk_api**, **Bots Long Poll API**, **SQLite** и переменные окружения через **.env**.

## Что умеет текущая версия

- Слушает входящие сообщения сообщества VK через Long Poll.
- Реагирует на триггеры: `старт`, `start`, `начать`, `привет`.
- Отправляет сообщения по кнопкам:
  - «Что входит»
  - «Как оплатить»
  - «Я оплатил»
  - «Партнёры»
  - «Задать вопрос»
- Сохраняет пользователей в локальную БД `database.db` при первом сообщении.
- Создаёт заявки на кнопку «Как оплатить», переводит в `paid` на кнопку «Я оплатил».
- Принимает скрин оплаты (вложения `photo`/`doc`) и отмечает факт получения в заявке.
- Отправляет пользователю ссылку в закрытый клуб после подтверждения админом `/approve {vk_id}`.
- Поддерживает админ-команды:
  - `/users`
  - `/requests`
  - `/pending`
  - `/approve {vk_id}`
  - `/stats`

## Структура проекта

- `main.py` — запуск бота, обработка событий Long Poll и админ-команд.
- `db.py` — работа с SQLite (`users`, `requests`).
- `texts.py` — все тексты бота.
- `keyboards.py` — VK-клавиатура.
- `.env.example` — пример переменных окружения.
- `requirements.txt` — зависимости.

## 1) Как создать токен сообщества VK

1. Откройте сообщество VK, где будет работать бот.
2. Перейдите: **Управление → Работа с API**.
3. Во вкладке **Ключи доступа** создайте ключ типа сообщества.
4. Выдайте права минимум на `messages`.
5. Скопируйте токен и сохраните его в `.env` как `VK_GROUP_TOKEN`.

## 2) Как включить Bots Long Poll

1. В сообществе перейдите: **Управление → Работа с API → Long Poll API**.
2. Включите Long Poll API.
3. Во вкладке **Типы событий** включите минимум событие **Входящее сообщение** (`message_new`).
4. Узнайте ID сообщества (число) и добавьте его в `.env` как `VK_GROUP_ID`.

## 3) Администратор

Добавьте в `.env` ID администратора сообщества и ссылку на закрытый клуб:

```env
ADMIN_ID=123456789
CLUB_INVITE_LINK=https://vk.me/join/...
```

Этот пользователь сможет выполнять админ-команды внутри диалога бота:

- `/users` — количество пользователей.
- `/requests` — список заявок в формате `vk_id | status | receipt_received | created_at`.
- `/pending` — заявки в `paid`, которые ещё не подтверждены.
- `/approve {vk_id}` — отметить последнюю заявку пользователя как `approved`, записать время подтверждения и отправить ссылку.
- `/stats` — общая статистика по пользователям и заявкам.

## 4) Процесс оплаты

1. Пользователь нажимает кнопку **«Как оплатить»** → бот создаёт заявку `new`.
2. Пользователь нажимает **«Я оплатил»** → заявка переводится в `paid`.
3. Пользователь отправляет скрин оплаты (`photo`/`doc`) → бот ставит `receipt_received = 1`.
4. Админ подтверждает оплату командой `/approve vk_id`.
5. Бот отправляет пользователю приветствие и `CLUB_INVITE_LINK` (доступ на 30 дней).

## 5) База данных

Бот использует SQLite-файл `database.db` (создаётся автоматически).

### Таблица `users`

- `id` — `INTEGER PRIMARY KEY AUTOINCREMENT`
- `vk_id` — `INTEGER UNIQUE`
- `first_name` — `TEXT`
- `last_name` — `TEXT`
- `created_at` — `TEXT`

### Таблица `requests`

- `id` — `INTEGER PRIMARY KEY AUTOINCREMENT`
- `vk_id` — `INTEGER`
- `status` — `TEXT` (`new` / `paid` / `approved`)
- `created_at` — `TEXT`
- `receipt_received` — `INTEGER` (`0` / `1`)
- `approved_at` — `TEXT`

При старте бота выполняется безопасная миграция: если новых колонок ещё нет, они добавляются через `ALTER TABLE`.

## 6) Установка и запуск

```bash
python3 -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
```

Заполните `.env`:

```env
VK_GROUP_TOKEN=ваш_токен_сообщества
VK_GROUP_ID=123456789
ADMIN_ID=123456789
CLUB_INVITE_LINK=https://vk.me/join/...
```

Запуск:

```bash
python main.py
```

## Примечания

- Бот работает через Long Poll (без webhook).
- Платежи не автоматизированы: подтверждение выполняет администратор.
- Для отправки сообщений используется `random_id`, как требует `messages.send`.
- Не храните реальные токены в репозитории.
